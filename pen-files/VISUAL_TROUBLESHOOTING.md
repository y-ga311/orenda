# 見た目が反映されないときの分析（Pencil / pencil-home.pen）

## よくある原因

### 1. **Cursor で保存した `.pen` と Pencil の内容がずれている**
Pencil は別バッファを開いたままにすることがあります。

- **対処**: Pencil で `pencil-home.pen` を**いったん閉じて開き直す**、または **Revert / ディスクから再読み込み**。

### 2. **画像の相対パスが Pencil から解決できない**
`../source-images/button.png` は **`.pen` があるフォルダからの相対**で正しいはずですが、  
環境によっては **1 つ上のフォルダを参照するパスが読み込まれない**ことがあります。

- **対処（ボタン）**: `pen-files/images/button.png` を用意し、`.pen` 内の URL を **`images/button.png`** に変更済みです（`.pen` と同じ `pen-files` 配下）。

### 3. **Dropbox の同期・競合**
同期のタイミングで、エディタが古いファイルを読んでいる可能性があります。

- **対処**: Dropbox の同期完了を待つ、競合ファイルがないか確認。

### 4. **見ているフレームが違う**
キャンバス上に **複数の 390×844 画面**がある場合、**Home Screen（`JHI7R`）** を選んでいるか確認してください。

### 5. **ログインボタンは「一番下の細い帯」**
レイアウトは **ロゴ → キャラ（縦に伸びる）→ ログインボタン（高さ 56）** です。  
キャラが大きいと、ボタンは**画面の一番下**に来ます。見失いがちです。

### 6. **ボタン画像が透明部分が多い**
`button.png` の**装飾が細い**と、`fill` の切り取りで目立たないことがあります。

- **対処**: `kzs9a` の `fill.mode` を **`fit`** に変えてみる、または高さを **60〜64** にする。

### 7. **HTML デモで確認する**
Pencil ではなく **ブラウザ**で UI を確認する場合:

```bash
cd （Orenda フォルダ）
python3 run_home_demo.py
```

`home-interactive.html` は `source-images/button.png` を参照します（ルート直下のデモ用）。
