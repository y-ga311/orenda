# pencil-home 要件定義メモ

## 概要

`pencil-home.pen` のデザインをもとに、学生向けスマートフォン用サービスを実装する。

このドキュメントは要件定義の作業メモとして扱い、今後の検討内容を随時加筆・修正していく。

## 想定ユーザー

- 学生のみが利用する
- 主にスマートフォンから利用する
- App Store / Google Play で一般公開せず、限定配布で利用してもらう

## サービスの目的

- 学生の勉強管理を支援する
- 学生が自分の勉強状況を記録・確認できるようにする
- 他の学生の勉強状況を共有することで、学習継続のきっかけを作る

## 主要機能

- 勉強時間の管理
- 問題取り組み
- 他の学生の勉強状態の共有
- カードコレクション

## 技術方針

- 実装言語は TypeScript を基本とする
- アプリ形態は PWA を前提とする
- フロントエンドは React / Next.js を候補とする
- UI 実装には Tailwind CSS を候補とする
- データベースには Supabase を使用する

## 配布方針

- URL または QR コードで配布する
- 学校ポータル、招待リンク、限定公開 URL などからアクセスさせる
- App Store / Google Play への公開は前提にしない

## 認証・利用制限

- 学生のみが利用できる仕組みを用意する
- 認証方式は Supabase の `students` テーブルに登録されている `gakusei_id` と `gakusei_password` を使用する
- 学生確認は、事前に ID と password を配布しているため追加対応しない
- 教員向け機能は本アプリには含めず、別アプリとして作成する
- 運用管理画面は本アプリには含めず、別アプリとして作成する

## プロフィール設定

- 初回ログイン時にプロフィール設定画面を表示する
- ニックネームは `students` テーブルに保存する
- アイコンはユーザーアップロードではなく、アプリ側で用意した固定アイコンから選択する
- 固定アイコンは `JHI7R` のプロフィール設定画面に表示されている9つを採用する
- `students` テーブルには画像データを保存せず、選択されたアイコンIDのみを保存する
- アイコンIDは `pixel01` から `pixel09` までを使用する

## スマートフォン対応

- スマートフォン画面を最優先に設計する
- ホーム画面追加に対応する
- 必要に応じて PWA 対応を行う
- 通知、カメラ、位置情報などの端末機能は必要性を確認してから検討する

## 将来的なアプリ化

- まずは Web アプリ / PWA として実装する
- ネイティブアプリ化が必要になった場合は Capacitor などを検討する

## Supabase 方針

- データベースには Supabase を使用する
- Supabase の `students` テーブルに登録されている `gakusei_id` と `gakusei_password` でログインする
- 個人情報は Supabase 内に保存済みのデータを使用する
- 本アプリ側で新たな個人情報保存方針は設けない
- 国家試験までの日数は、`students.class` と `national_exam_schedules.class_name` を照合し、`exam_date` と日本時間の本日から計算する
- 料金プランは無料枠を前提とする
- バックアップと削除方針は GitHub 管理を前提とする
